    Backtrace(moderate)¶

    :::warning 题目：

    backtrace 是一个存放于栈上用于指示错误发生位置的函数调用列表。

    在kernel/printf.c中实现名为**<font style="background-color:rgb(247, 247, 247);">backtrace()</font>**的函数。在**<font style="background-color:rgb(247, 247, 247);">sys_sleep</font>**中插入一个对此函数的调用，然后运行**<font style="background-color:rgb(247, 247, 247);">bttest</font>**，它将会调用**<font style="background-color:rgb(247, 247, 247);">sys_sleep</font>**。你的输出应该如下所示：

    backtrace:

    0x0000000080002cda

    0x0000000080002bb6

    0x0000000080002898

    :::

    :::success 解决方案：

    编译器在每一个栈帧中放置一个帧指针（frame pointer）保存调用者帧指针的地址。通过使用这些帧指针我们可以遍历栈，并在每个栈帧中打印保存的返回地址。

    tips:

    • 在 kernel/defs.h 中添加**<font style="background-color:rgb(247, 247, 247);">backtrace</font>**的原型，那样你就能在**<font style="background-color:rgb(247, 247, 247);">sys_sleep</font>**中引用**<font style="background-color:rgb(247, 247, 247);">backtrace</font>**
    • GCC 编译器将当前正在执行的函数的帧指针保存在**<font style="background-color:rgb(247, 247, 247);">s0</font>**寄存器，将下面的函数添加到kernel/riscv.h（并在**<font style="color:rgb(51, 51, 51);background-color:rgb(247, 247, 247);">backtrace</font>**中调用此函数来读取当前的帧指针。这个函数使用内联汇编来读取**<font style="color:rgb(51, 51, 51);background-color:rgb(247, 247, 247);">s0</font>**）
    • XV6 在内核中以页面对齐的地址为每个进程的栈分配一个页面。你可以通过**<font style="background-color:rgb(247, 247, 247);">PGROUNDDOWN(fp)</font>**和**<font style="background-color:rgb(247, 247, 247);">PGROUNDUP(fp)</font>**（参见kernel/riscv.h）来计算栈页面的顶部和底部地址。这些数字对于**<font style="background-color:rgb(247, 247, 247);">backtrace</font>**终止循环是有帮助的。

    :::

r_fp()
{
  uint64 x;
  asm volatile("mv %0, s0" : "=r" (x) );
  return x;
}

    void backtrace()
{
  printf("backtrace:\n");
  uint64 this_fp = r_fp();
  backtrace_help(&this_fp);
  return;
}
void backtrace_help(uint64 *this_fp)
{
  // 判断是否超出栈，若超出栈，停止
  if ((PGROUNDUP(*this_fp) - PGROUNDDOWN(*this_fp)) != PGSIZE)
  {return;}
  // 返回地址保存在 -8 偏移的位置
  printf("%p\n", *(uint64 *)(*this_fp - 8));
  // 前一个帧指针保存在-16偏移的位置
  uint64 last_fp = *this_fp - 16;
  backtrace_help(&(*(uint64 *)last_fp)); // 注意这里的指针的转换为什么是必要的
  return;
}

    Alarm(Hard)¶

    :::warning 题目：

    你将向 XV6 添加一个特性，在进程使用 CPU 的时间内，XV6 定期向进程发出警报。这对于那些希望限制 CPU 占用时间的的进程，或者计算的同时执行某些周期性操作的进程很有用。

    更普遍的来说，你将实现用户级中断 / 故障处理程序的一种初级形式。

    例如，你可以在应用程序中使用类似的一些东西处理页面故障。

    :::

    :::warning 1. 应当添加一个新的**<font style="color:rgb(51, 51, 51);background-color:rgb(247, 247, 247);">sigalarm(interval, handler)</font>**系统调用 2. 如果一个程序调用了**<font style="color:rgb(51, 51, 51);background-color:rgb(247, 247, 247);">sigalarm(n, fn)</font>**，那么程序每消耗了 CPU 时间达到 n 个“滴答”，内核应当使应用程序函数**<font style="color:rgb(51, 51, 51);background-color:rgb(247, 247, 247);">fn</font>**被调用。 3. 当**<font style="color:rgb(51, 51, 51);background-color:rgb(247, 247, 247);">fn</font>**返回时，应用应从它离开的地方恢复执行。在 XV6 中，一个滴答是一段相当任意的时间单元，取决于硬件计时器生成中断的频率。 4. 如果一个程序调用了**<font style="color:rgb(51, 51, 51);background-color:rgb(247, 247, 247);">sigalarm(0, 0)</font>**，系统应当停止生成周期性的报警调用。

    :::

    :::

    void usertrap(void)
{
    int which_dev = 0;
    if ((r_sstatus() & SSTATUS_SPP) != 0)
        panic("usertrap: not from user mode");
    // send interrupts and exceptions to kerneltrap(),
    // since we're now in the kernel.
    w_stvec((uint64)kernelvec);
    struct proc *p = myproc();
    // save user program counter.
    p->trapframe->epc = r_sepc();
    /** 判断是什么原因调用了usertrap函数*/ 
    // 是否是系统调用
    if (r_scause() == 8)
    {
        // system call
        if (p->killed)
            exit(-1);
        // sepc points to the ecall instruction,
        // but we want to return to the next instruction.
        p->trapframe->epc += 4;
        // an interrupt will change sstatus &c registers,
        // so don't enable until done with those registers.
        // 关闭中断
        intr_on();

        syscall();
    }
    // 是否是设备中断
    else if ((which_dev = devintr()) != 0)
    {
        // ok
    }
    // 处理意外的陷阱
    else
    {
        printf("usertrap(): unexpected scause %p pid=%d\n", r_scause(), p->pid);
        printf("            sepc=%p stval=%p\n", r_sepc(), r_stval());
        p->killed = 1;
    }
    if (p->killed)
        exit(-1);
    // give up the CPU if this is a timer interrupt.
    // 如果这是一个定时器中断，则放弃 CPU。
    if (which_dev == 2){

        #ifdef lab_traps
        if (p->alarm_ticks != 0 && p->alarm_status != 1)
        {
            p->total_ticks++;
            if (p->alarm_ticks == p->total_ticks)
            {
                p->total_ticks = 0;
                // void *handler() = (void *)p->handler;
                //void (*handler)(void) = (void (*)(void))p->handler;
                //handler();
                // 需要保存寄存器内容，我们要在执行完报警程序后回到原本的代码执行处
                memmove(p->alarm_trapframe, p->trapframe, sizeof(struct trapframe));
                // 正常情况下，中断恢复执行epc指向的代码，也就是恢复原本代码，但我们这个lab中
                // 通过更改epc来改变中断后执行的函数，最后再通过 sigreturn 进行恢复
                // 这就是为什么在上面我们保存了寄存器的内容
                p->trapframe->epc = p->handler; 
                p->alarm_status = 1;
            }
        }
        #endif

        yield();
    }
    usertrapret();
}

    :::warning + 在跳转执行特定的处理函数后，我们需要返回到原本的代码值 + 这个特定的处理函数需要配合 sigreturn 一起使用，处理函数处理完后通过 sigreturn 来返回到原本的代码
