    Lecture 21：竞争条件 ¶

    今天，我们探索在单个进程中（使用线程）并发性可能引入的复杂性。特别是竞争条件，当多个线程访问同一内存位置且至少其中一个访问是写入时，就会发生竞争条件。

    竞争条件会导致程序中的未定义行为，并经常导致难以找到的错误和看似奇怪的数据损坏。

    为了避免程序中的竞争条件，我们 需要 添加同步机制，通过不同线程之间的有序访问，使它们不能以未定义的方式交错。这在下节课中生深入讨论。

    void threadfunc(unsigned* x) {
    for (int i = 0; i != 10000000; ++i) {
        *x += 1;
    }
}
int main() {
    //四个线程
    std::thread th[4];
    unsigned n = 0;
    for (int i = 0; i != 4; ++i) {
        th[i] = std::thread(threadfunc, &n);
    }
    for (int i = 0; i != 4; ++i) {
        th[i].join();//join 方法会阻塞调用它的线程（即主线程），直到对应的子线程执行完毕
    }
    printf("%u\n", n);
}
    • 四个线程同时运行函数 threadfunc() 。每个线程中， threadfunc 会将一个共享变量递增 1000 万次。总共有四个线程在递增，而变量的初始值为零。在所有递增线程执行完毕后，变量的最终值应该是多少？

    4000 万似乎是一个合理的答案，但实际运行结果却非常奇怪，打印出各种各样的值，如 15285711，且每次值都不同。

    • 为什么呢？

    addl 指令本身存在竞态条件！（术语“竞态”指的是两个线程“竞争”访问、修改和更新单个内存位置中的值。）

    之前，我们一直认为 x86 指令是不可分割和原子的。实际上，它们并不是，它们的非原子性在多处理器环境中显现出来。

    • 在处理器硬件内部， addl $1, (%rdi) 实际上被实现为三个单独的“微操作”指令： 1. movl (%rdi), %temp (load)（加载） 2. addl $1, %temp (add)（添加） 3. movl %temp, (%rdi) (store)（存储）

    所以，想象两个线程同时执行 addl $1, (%rdi) 。每个线程从内存中加载相同的 (%rdi) 值，然后在各自的临时寄存器中加一，然后将相同的值写回内存中的 (%rdi) 。每个线程对内存的最后一次写入将用相同的值覆盖彼此，一个增量实际上会丢失。

    :::tips incr-basic 的优化版本呢？
    开启编译器优化后，编译器可以优化掉循环中的 threadfunc() 。它会识别到循环只是简单地对共享变量进行 1000 万次递增操作，因此将循环转换为一条带有立即值 1000 万的 addl 指令。

    编译后的程序似乎可以运行并产生正确的输出，但如果我们将其放在一个循环中（ ./incr-basic | uniq ），最终会打印出 30000000 或甚至 20000000 这样的值。这是因为即使只有一次对 1000 万的加法操作，不同线程执行的四次加法之间仍然存在竞争条件。

    :::

    在 C/C++ 中，如果没有适当的同步，从不同的线程访问共享内存是未定义的行为，除非所有访问都是读取操作。

    • 再看一个示例程序

    其中两个线程都试图在内存中递增一个初始值为 7 的整数 x 。每个线程需要（i）从内存中加载值；（ii）递增它；和（iii）将递增后的值写回内存。

    在实际处理器中，这些步骤可能以微码指令的形式发生，但在这里我们展示的是在寄存器 %tmp 上操作的 x86-64 汇编指令。

    1. 左图：我们看到 T1 和 T2 线程的交错，这种交错是不好的！它丢失了一个递增操作，最终得到的值是 x = 8 ，尽管 x 已经递增了两次。这个问题的原因是 T2 的 movl (&x), %tmp 指令在 T1 的递增操作（ addl ）之前运行，所以运行 T2 的处理器将一个过时的 x 值存储在其缓存和寄存器中，并对该值进行递增。两个线程都将值 x = 8 写回内存。
    2. 右图：这种顺序是合适的，线程之间没有交错地运行操作，可以得到正确的结果。

    结论：我们的程序的结果若依赖于线程的随机交错执行将会很糟糕，所以我们最好同步线程，限制执行到“好”的交错。

    这就是同步的基本规则：如果两个或多个线程可以同时访问一个对象，并且至少有一个访问是写操作，则可能发生竞争条件，需要进行同步。
