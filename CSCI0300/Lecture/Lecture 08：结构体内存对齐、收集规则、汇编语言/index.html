    Lecture 08：结构体内存对齐、收集规则、汇编语言¶

    结构体中的内存对齐与malloc返回指针内存问题，初步开始汇编

    • 内存对齐——结构体内存对齐
    • 结构体的变量在内存分配的时候，需要按照变量在结构体内的顺序依次进行内存分配，编译器不会自动优化内存分配，所以因为这个和内存对齐，一般结构体分配的内存比实际需要的内存更多。
    • 对于结构体，可以通过调整结构体内变量声明的顺序来节约空间

    malloc 规则：成功调用 malloc 返回的指针对任何类型都是对齐的。

    1. 这个规则有一些重要的后果：这意味着 malloc() 必须返回对齐于最大对齐的指针，在 x86-64 Linux 上是 16 字节。换句话说，malloc 返回的任何指针都指向一个地址，该地址是 16 的倍数。

    2. 汇编语言

    从 C 代码到指令

    如何阅读一个汇编文件

    1. 只关注你关心的指示，最初忽略其他任何事情。
    2. 从返回语句开始逆向工作。

    如何从c源文件获取汇编

    <font style="color:rgb(33, 37, 41);"></font><font style="color:rgb(232, 62, 140);">gcc -S testasm.c</font>

    示例：

    .file "testasm.c"
    .text
    .globl  _Z1fiii
    .type   _Z1fiii, @function
_Z1fiii:
.LFB0:
    cmpl    %edx, %esi
    je  .L3
    movl    %esi, %eax
    ret
.L3:
    movl    %edi, %eax
    ret
.LFE0:
    .size   _Z1fiii, .-_Z1fiii
    .ident  "GCC: (Ubuntu 7.4.0-1ubuntu1~18.04.1) 7.4.0"
    .section        .note.GNU-stack,"",@progbits
    1. 这里有许多有效的注释行。所有以点号开头的行（例如， .file 或 .ident ）都属于这种类型：它们构成指令性语句（也称为“指示”或“指令”）而不是执行指令。这些指令性语句主要是给安装器的的指令，告诉编译器应该如何处理接下来的代码，但对于理解代码的逻辑流程通常不是必须的。
    2. 然而，所有以点开头以冒号结尾的指令行都很重要：它们构成标签，对于控制流指令（例如， .L3: ）很重要。这在程序的流控制中是非常重要的。因为标签是用来标记程序中的位置，便于控制流指令（如跳转）可以引用，从而改变程序的执行路径。
