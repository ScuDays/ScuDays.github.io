    Lecture 20：多进程、线程¶

    我们了解了并发性和并行性之间的区别，并发性 concurrency 是指在计算机上同时激活多个进程，而并行性parallelism是指计算机在任何给定时间点实际上执行多件事。
    最后，我们开始研究线程，它是并发和并行性的另一种抽象，它允许单个进程使用自己的堆栈运行多个并行指令流（“线程”），但同时共享所有其他内存。

    并发性与并行性 Concurrency vs. Parallelism¶

    • 并发性

    这个概念涉及多个进程同时在计算机上处于活动状态，称为多进程，并构成一种并发形式。并发意味着多个进程同时处于活动状态。但多个进程并不是真的同时进行，而是通过时间分片的方式交替进行。在单核处理器上，操作系统通过快速切换任务，使得看起来像是多个任务在同时进行。

    • 并行性

    并行性的概念与并发性有微妙的不同：它指的是在不同的处理器上同时实际执行来自多个进程（或线程）的指令。为了实现并行性，您需要多个同时处于活动状态的处理器。但并行执行也是并发的，因为多个进程（或线程）仍然同时处于活动状态。

    线程¶

    管道和 IPC 通常允许隔离地址空间中的进程以受限和内核介导的方式进行通信。但有时你希望多个并发执行单元在真正的共享内存上工作（例如，因为数据很大，fork 的成本很高;或者因为你正在实现一个应用程序，真正共享数据跨并行请求）。为此，操作系统提供了线程，这些线程像进程一样并发执行，但共享其父进程的单个虚拟地址空间。

    下面是父进程和子进程共享的内容与同一进程中的两个线程共享的内容之间的比较：

    • 一个进程可以包含多个线程。同一进程中的所有线程共享相同的虚拟地址空间和文件描述符表。但是，每个线程必须有自己的一组寄存器和堆栈。（想想如果两个线程使用相同的堆栈并在不同的处理器上并行运行，那么所有的事情都会出错！
    • 线程允许在单个进程内实现并发：即使一个线程在等待某些事件（例如来自网络或硬盘的 I/O）时被阻塞，其他线程也可以继续执行。在具有多个处理器的计算机上，线程还允许并行执行多个指令流。
    • 同一进程中的所有线程共享相同的虚拟地址空间和文件描述符表，但每个线程都有自己的一组寄存器和堆栈。
    • 每个进程至少有一个“主”线程——因此，我们的进程有一个线程，而不是零个线程。对于多线程进程，内核为每个线程存储一组寄存器，而不是为每个进程存储一组寄存器。这是必要的，因为内核需要能够独立运行处理器上的每个线程，并再次挂起它以让其他线程运行。
    • 来看 一个使用线程的示例程序
    void threadfunc(unsigned* x) {
    for (int i = 0; i != 10000000; ++i) {
        *x += 1;
    }
}
int main() {
    //四个线程
    std::thread th[4];
    unsigned n = 0;
    for (int i = 0; i != 4; ++i) {
        th[i] = std::thread(threadfunc, &n);
    }
    for (int i = 0; i != 4; ++i) {
        th[i].join();//join 方法会阻塞调用它的线程（即主线程），直到对应的子线程执行完毕
    }
    printf("%u\n", n);
}
    • 四个线程同时运行函数 threadfunc() 。每个线程中， threadfunc 会将一个共享变量递增 1000 万次。总共有四个线程在递增，而变量的初始值为零。在所有递增线程执行完毕后，变量的最终值应该是多少？

    4000 万似乎是一个合理的答案，但实际运行结果却非常奇怪，打印出各种各样的值，如 15285711，且每次值都不同。—— 下一讲讲解 —— 竞争条件
