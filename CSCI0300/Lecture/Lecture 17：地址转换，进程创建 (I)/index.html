    Lecture 17：地址转换，进程创建 (I) ¶

    1. 虚拟地址转换：x86-64 架构使用四级页面表。在一个虚拟地址中，只有前 48 位是有意义的。我们有 9 位用于索引每个页面表级别，12 位用于偏移。每个页面表在每个层级都有 512 个条目。这也是为什么虚拟地址中的索引每个都是 9 位长的原因！我们需要 9 位来从每个页面表块中选择一个出 512 个条目。
    2. 进程生命周期：在 WeensyOS 中，内核在启动时启动四个进程，但（至少在项目 4 的步骤 5 之前）用户空间进程无法启动另一个用户空间进程。一个现实的操作系统显然需要能够这样做。
    3. 进程创建：许多基于 Unix 的操作系统（包括 Linux、BSD 系列操作系统和 Mac OS X）都使用名为 fork() 的系统调用来创建进程。fork() 的效果是克隆用户空间进程。
    4. fork() 系统调用：fork() 的效果是克隆用户空间进程，会导致多个进程从相同的内存映射开始。

    Process Lifecycle 进程生命周期 ¶

    • 进程是我们在计算机上运行程序的方式，我们的计算机通常使用多个进程来完成任务。例如，一个简单的终端命令，如 ls 或 grep ，每个都会运行一个新的进程，产生一些输出然后退出。
    • 在 WeensyOS 中，内核在启动时启动了四个进程，但是（至少在第 4 个项目的第 5 步之前），用户空间进程无法启动另一个用户空间进程。一个真实的操作系统显然需要能够这样做。

    Process Creation 进程创建 ¶

    fork() 创建进程 ¶

    许多基于 Unix 的操作系统，包括 Linux、BSD 系列操作系统和 Mac OS X，使用一个名为 fork() 的系统调用来创建进程。

    fork 在使用了近 50 年后仍然引起争议，它并不是创建进程的唯一方式（例如，Windows 采用了不同的方法）。但这是数百万台计算机和设备创建进程的方式！

    • The fork() system call

    fork() 系统调用的作用是克隆一个用户空间进程。

    什么叫克隆用户空间进程呢？请看例子：

    这个程序使用 fork（），从调用 fork() 的那一行，程序同时开始执行了一个新的进程。这就意味着，fork() 之后的代码，程序就会有两个进程（一个父进程和一个新生成的子进程）在运行。

    #include "helpers.hh"
int main() {
    pid_t p1 = fork();
    assert(p1 >= 0);
    printf("Hello from pid %d\n", getpid());
}

    所以会有多少个 “Hello from pid %d\n” 被打印呢？

    很明显 会有两个消息被打印出来 —— 父进程和子进程各一个。

    为什么呢？

    这是因为对 fork() 的调用进入了内核，内核会克隆进程，并在两个克隆体中继续执行用户空间代码。

    两个进程都会执行程序的剩余部分，因此都会执行 printf 函数调用。

    需要注意的是，这两个进程具有不同的进程 ID，可通过 getpid() 系统调用返回的不同值来看。

    :::color3 如何判断当前进程是子线程还是父线程？

    通过 fork（）的返回值： 从 fork() 返回的值取决于它是返回到父级还是返回到子级 - 每次成功调用 fork() 都会返回两次：

    1. In the parent process, the return value is the new process's PID.
      在父进程中，fork（）的返回值是新进程的 PID。
    2. In the child process, the return value is 0.
      在子进程中，fork（）的返回值是 0。
    3. 负的返回值表示 fork() 失败，没有创建子进程。

    :::

    fork () 需要克隆的资源¶

    内核克隆一个分叉的子进程，这个子进程与其父进程的许多资源相同，因此操作系统内核需要从父进程复制各种信息。

    需要复制的信息包括：

    • 处理器描述符（一个内核数据结构，保存有关进程的信息；在 WeensyOS 中，当前进程的描述符为 current ，i th 进程的描述符为 proc[i] ）。进程描述符还包括进程的当前寄存器值；两个进程在进入用户空间执行时，除了 %rax 之外，寄存器内容完全相同。
    • 虚拟内存映射，即页表（尽管映射需要由新的物理页面支持）。
    • 文件描述符和网络连接（WeensyOS 中没有文件 / 套接字）。

    多重 fork() ¶

    请记住，对于父进程和子进程来说，执行在同一个程序中继续进行（尽管它们的执行可能会分叉）。

    如果子进程再次进行分叉，它可以创建更多的进程（参见 fork2.cc ，最终总共有四个进程）。

    请看例子：

    C
    1

    int main() {
    printf("Hello from initial pid %d\n", getpid());
    pid_t p1 = fork();
    assert(p1 >= 0);
    pid_t p2  = fork();
    assert(p2 >= 0);
    printf("Hello from final pid %d\n", getpid());
}
    这个会打印多少次消息呢？

    5 行。第一个 printf() 打印一行，只在父进程中，然后第二个 printf() 将运行四次，分别在每个进程中（父进程 +2 个子进程 +1 个孙子进程）。
