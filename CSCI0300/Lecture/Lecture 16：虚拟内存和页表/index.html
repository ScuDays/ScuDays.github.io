    研究如何保护内存：通过使用页面权限来控制用户空间程序，防止其修改内核内存，并通过改变内存映射，使得内核内存对用户空间进程不可用。

    并简单的介绍了页表的机制。

    计算机使用页表实现虚拟内存，页表是一种映射表，用于将虚拟地址（用户空间程序使用的地址）转换为物理地址（指计算机DRAM芯片中的真实内存地址）。x86-64架构中的页表采用了巧妙的四级设计，可以将其分块成页面大小的单元（因此页表本身可以存储在物理内存页面中），并且具有高级截断功能，避免浪费空间存储虚拟地址的转换结果（即没有对应物理页面的虚拟地址）

    Virtual Memory 虚拟内存 ¶

    为什么需要虚拟内存¶

    我们之前使用内存保护来隔离内核内存和用户空间进程，防止用户空间进程写入内核内存的攻击。

    但这还不够 - 我们还需要防止用户空间进程访问其他用户空间进程的内存！

    Eve 和 Alice 的例子

    1. 如果 Eve 成功猜测出 Alice 的内存地址，她可以读取和修改 Alice 的数据。
    2. 如果 Alice 和 Eve 的进程意外使用了相同的地址，它们将破坏彼此的内存；程序员需要仔细选择非重叠的内存区域来运行它们的进程——浪费精力

    Alice 的进程代码存储在堆上，若 Eve 可以访问 Alice 的内存地址，则可以让 Alice 帮 Eve 运算，或者强制让 Alice进入一个无限循环，永久禁用她的进程。（危险！！！）

    所以我们需要虚拟内存！¶

    虚拟内存背后的基本思想是为每个用户空间进程创建一个假象，使其在计算机上独立运行，并可以访问计算机的全部内存。换句话说，我们希望给不同的进程提供不同的实际内存视图。

    但这样的效果，要怎么样实现呢？—— 使用页表

    页表¶

    页面表是让我们能够将虚拟地址转换为物理地址的东西。每个用户空间进程都有自己的页面表，它用于执行该转换。

    页表设计的三个尝试：

    - **<font style="color:rgb(6, 6, 7);">尝试1</font>**<font style="color:rgb(6, 6, 7);">：直接的字节级查找表，每个字节都有一个对应的页表项，但这种方法非常耗内存。</font>
- **<font style="color:rgb(6, 6, 7);">尝试2</font>**<font style="color:rgb(6, 6, 7);">：只存储每页的条目，使用虚拟地址的低12位作为页内偏移，减少了页表的大小。</font>
- **<font style="color:rgb(6, 6, 7);">尝试3</font>**<font style="color:rgb(6, 6, 7);">：只存储物理地址，进一步减少内存使用，但仍然占用大量空间。</font>

    x86-64页表和地址转换：实际使用的页表结构结合了上述尝试，并增加了一个巧妙的树状结构。x86-64使用四级页表结构，减少了所需空间并允许稀疏树结构。

    x86-64页表示例（WeensyOS 页表）¶

    WeensyOS 页表 示例

    • 这种结构意味着有 1 个 L4 页表，最多 512 个 L3 页表（每个 512 个 L4 PT 条目可以指向一个单独的 L3 PT 页），最多 512 个 2 L2 页表和最多 512 个 3 L1 页表。
    • 但实际上，数量并没有那么多，如图所示。使用所有 512 个 3 L1 页表将允许寻址 512 个 3 × 512 = 512 个 4 ≈ 680 亿个页面。每个页面的大小为 4096 字节，这将覆盖 256 TB 的可寻址内存；页表本身的大小将为 512 GB。大多数程序只需要其中一小部分空间，并且通过在高级别（例如 L3 或 L4）保留页表条目为空，整个页表结构的子树就不需要存在。
    • 多级页面表结构（x86-64 使用四级；x86-32 使用两级）可看作是一棵树。多级可以减少所需的空间，因为当我们查找给定虚拟地址的物理页面时，我们可能会访问树的一个分支，告诉我们实际上没有有效的物理页面供我们访问。在这种情况下，我们只需停止搜索。因此，多级意味着我们可以拥有一个稀疏的树。
    • L1页表项是特殊的，因为它们提供了将物理地址转换为虚拟地址的实际信息。L1页表项（PTE）不存储另一个页表的地址，而是包含了将给定的虚拟地址（VA）转换为的部分物理地址（PA），以及控制页面访问权限的权限位（PTE_P、PTE_W和PTE_U；以及其他我们在本课程中不详细介绍的位）。
    • 注意，每个进程都有自己的页表
    • 在 WeensyOS 中，内核代码包含一个全局数组 ptable ，其槽包含每个进程的顶级（L4）页表的地址。这样，内核可以定位并修改每个进程的页表。
