    Lecture 09：汇编语言¶

    汇编通过操作寄存器实现目的，寄存器是机器中速度最快的存储类型。

    汇编指令的三种基本类型：计算型、数据移动型和控制流型。

    使用编译器的一个原因：它们能够巧妙地将我们的程序简化为最快的指令序列。

    :::info + x86-64 有 14 个通用寄存器和几个特殊用途寄存器。

    :::

    下表列出了所有基本寄存器，特殊用途寄存器用黄色突出显示。（无需记忆）

    • 注意：寄存器和内存不同！！！

    与主存储器（RAM）不同 - 当我们讨论 C/C++ 程序中的内存时，我们所考虑的是主存储器 —— 寄存器没有地址！没有地址值，如果转换为指针并进行解引用，将返回 %rax 寄存器的内容。寄存器存在于与内存分开的世界中，我们需要特殊指令来在寄存器和内存之间传输数据。

    :::info + 寄存器的命名问题

    :::

    由于发展时间太长，命名产生了分化，同一寄存器有很多不同的名字。用%zzz代表zzz这个寄存器。

    :::info 汇编指令通常具有后缀，指示它们正在操作的输入数据大小和寄存器宽度。

    :::

    **例如，(a set of move)一组“move”指令有助于将有符号和无符号的 8 位、16 位和 32 位数量从内存加载到寄存器中。 **

    • movzbl 将 8-bit quantitya (byte）移入 32 位寄存器（a longword; e.g., %eax)）并进行零扩展
    • movslq 将 32-bit quantity（longword）移入 64 位寄存器（quadword; e.g., %rax）并进行符号扩展

    上面的longword，指的不是c语言里的long，而是x86-64硬件术语里的long，32bit——4个字节，而不是8个字节。

    :::info + 汇编指令的三种基本类型

    :::

    1. 计算指令：

    这些指令对值进行计算，通常是存储在寄存器中的值。大多数指令具有零个或一个源操作数和一个源/目的操作数，源操作数在前。例如，指令 addq %rax, %rbx 执行计算 %rbx := %rbx + %rax

    1. 数据移动指令:

    这些指令在寄存器和内存之间移动数据，因此它们可以将一个寄存器中的值移动到另一个寄存器中，从内存移动到寄存器中，以及从寄存器返回到内存。几乎所有的移动指令都有一个源操作数和一个目的操作数; 源操作数首先出现。例如，movq %rax, %rbx 将 %rax 的内容复制到 %rbx ，因此执行赋值 %rbx = %rax 。

    1. 控制流指令:

    通常CPU 按照汇编代码中的顺序执行指令（一旦转换为字节，就是内存中的顺序）。控制流指令更改处理器执行的下一条指令（称为“指令指针”，存储在特殊寄存器 %rip 中）。有无条件分支（指令指针设置为新值），条件分支（如果条件为真，则指令指针设置为新值），以及函数调用和返回指令。

    一些指令似乎将计算和数据移动结合在一起。例如，给定 C 代码 int pi; … ++(*pi); ，编译器可能生成 incl (%rax) 而不是 movl (%rax), %ebx; incl %ebx; movl %ebx, (%rax) 。然而，处理器实际上将这些复杂指令分解为微小、更简单、不可见的指令，称为微码，因为简单指令可以更快地执行。复杂的 incl 指令实际上分为三个阶段：数据移动，然后计算，然后数据移动。当我们引入并行性时，这一点很重要。*
