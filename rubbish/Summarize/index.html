    声明和定义的区别¶

    :::color1 1. 声明（Declaration）: 告诉编译器某个变量或函数的存在。

    :::

    对于变量来说，声明让编译器知道变量的类型和名字。对于函数来说，声明包括函数的返回类型，它的名字，以及参数的类型和数量（也就是函数的原型）。声明不会分配内存（除了extern变量的声明之外，它不会为变量分配内存）。

    • 变量声明的例子：extern int a; 这告诉编译器变量a存在，并且是一个整型（int），但并不分配内存。
    • 函数声明的例子：int foo(int a, int b); 这告诉编译器存在一个名为foo的函数，它接收两个整型参数，并返回一个整型值。这就是函数的原型声明。

    :::color1 2. 定义（Definition）: 给某个变量或函数赋予确切的值。

    :::

    对于变量，定义是声明的一个扩展它不仅告诉编译器变量的类型和名称，还分配了内存。对于函数，定义包含了函数的实际代码。

    • 变量定义的例子：int a; 这不仅告诉编译器a是一个整型变量，还为其分配了内存。
    • 函数定义的例子：int foo(int a, int b) { return a + b; } 这提供了函数foo完整的代码实现。

    简而言之，声明通知编译器某些元素的性质，定义则负责实际创建和描述这些性质。

    一个简单的规则是：定义是声明，但声明不一定是定义。

    全局变量生命周期¶

    在C语言程序链接阶段，编译器会将所有的对象文件（即编译后的源文件，通常是.o文件）以及用到的库文件集合起来，形成一个单一的可执行文件。

    在这个过程中，所有的全局变量都会被分配空间，不论它们是否会被用到。

    这是静态链接的行为，而且这些全局变量的作用域是整个程序，它们在程序的整个生命周期内都会存在。

    FILE 和 LINE（c语言中预定义的宏）¶
    1. FILE 宏在编译时会被替换成一个字符串常量，内容是包含当前源代码文件名的字符串。这个宏经常用于错误消息中，以指示出错的代码所在的源文件。
    2. LINE 宏在编译时会被替换成一个整型常量，内容是当前源代码行号的数字。这同样经常用于错误处理中，以报告出错代码的具体位置。
    3. DATE: 当编译源代码时，这个宏会被替换成一个字符串，包含了文件被编译的日期，格式为“Mmm dd yyyy”，即“月份 日 年”的格式。
    4. TIME: 这个宏会被替换成一个字符串，包含了文件被编译的具体时间，格式为“hh:mm:ss”。

    函数指针¶

    函数指针定义：函数指针是指向函数的指针变量。因此“函数指针”本身首先应是指针变量，只不过该指针变量指向函数。这正如用指针变量可指向整型变量、字符型、数组一样，这里是指向函数。

    int (*fun)(int x)  //函数指针的定义，后面的参数对应你想要指向的函数
int (*fun)(int x，int y) //函数指针的定义

int (*fun)(int x，int y) //函数指针的定义
fun = &Function          //函数指针的赋值方式1
fun = Function           //函数指针的赋值方式2
x = (*fun)(x, y)             //函数指针的调用方式1
x = fun(x, y)                //函数指针的调用方式2
    jyy之前写的小工具，怎么写？¶

    :::color1 Linux常用命令行工具:sort

    :::

    下面是一些sort命令的使用示例：

    • 将文件中的行按照字典序排序：

    **sort filename**

    • 将排序结果逆序输出（比如，从高到低）：

    **sort -r filename**

    • 根据第二列的数字排序：

    **<font style="color:#000000;">sort -k 2n filename</font>**

    • 检查文件是否已排序，并不实际排序输出：

    **<font style="color:#000000;">sort -c filename</font>**

    • 排除重复的行：

    **<font style="color:#000000;">sort -u filename</font>**

    • 结合管道（pipe）使用sort对标准输出进行排序：

    **<font style="color:#000000;">cat filename | sort</font>**

    sort命令非常强大，配合其他文本处理工具（如awk, sed, grep等）一起使用，可以处理复杂的文本排序和数据提取任务。

    访问一个可能是野指针可能会产生的内存对齐问题(访问特定字节的东西，必须内存对齐)¶
    • 内存对齐问题通常是指在访问内存时，某些处理器要求数据的地址必须是某个数（通常是2的幂）的倍数。这意味着如果数据类型的自然对齐要求是4字节，那么它的数据地址应该是4的倍数。如果数据没有正确对齐，处理器可能会产生效率问题，或者在某些严格要求内存对齐的系统中，可能直接导致硬件异常。
    • 野指针是指那些没有被初始化，或者已经释放但仍被访问，或指向一个随机内存地址的指针。这些指针是危险的，因为它们可能指向任何地方，包括操作系统或其他程序的敏感区域。更糟糕的是，如果你尝试通过一个野指针访问或写入数据，可能会导致未定义行为，其中之一就是内存对齐问题。
    • 例如，假设你有一个类型为int的野指针，int类型在你的系统中需要4字节对齐，但这个野指针指向了一个不是4的倍数的地址，如0x10003。如果你尝试通过这个野指针来访问或者修改int值，那么可能会遇到对齐错误。
    void * 的运用，转其他类型，加减运算¶
    • void 在英文中作为名词的解释为 “空虚、空间、空隙”，而在 C 语言中，void 被翻译为”无类型”，相应的void * 为”无类型指针”。

    规则：

    1. void 指针可以指向任意类型的数据，就是说可以用任意类型的指针对 void 指针对 void 指针赋值

    例如：

    int *a；
void *p；
p=a；
    1. 但 void 指针转换为别的指针需要类型强制转换。

    例如：malloc()分配内存，返回的就是一个 void指针，我们需要(int )malloc(1024)来进行强制类型转换。

    1. ** void 指针不能进行算术运算如 p++ 或 p+=1 ，因为void 是无类型，那么每次算术运算时无法知道对几个字节进行操作*
    2. void* 指针经常用于分配 内存中
